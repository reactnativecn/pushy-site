# Pushy - 专业的热更新解决方案

<Callout type="info">
**现已全面支持鸿蒙系统以及 React Native 新架构！**
</Callout>

Pushy (npm 包名: [`react-native-update`](https://www.npmjs.com/package/react-native-update) <img src="https://badge.fury.io/js/react-native-update.svg" alt="npm version" style={{display: 'inline', verticalAlign: 'middle', margin: '0 4px'}} />) 是面向 React Native 应用提供的**专业热更新服务**，由 [React Native 中文网](https://reactnative.cn)独家研发并持续维护。我们致力于为中国开发者提供最快速、最稳定、最经济的热更新体验。

## 🚀 为什么选择 Pushy？

### 极致的更新体验

- **🎯 超小更新包**：基于 bsdiff/hdiff 算法的增量更新，通常只需**几十 KB**（相比其他全量更新服务的几十 MB，节省流量 99%+）
- **⚡️ 极速分发**：国内用户使用阿里云高速 CDN，国外用户智能分流至 Cloudflare，更新成功率高达 **99.9%+**
- **🔄 智能更新**：跨越多个版本时只需下载**一个更新包**，无需逐版本更新
- **🛡️ 安全可靠**：内置崩溃回滚机制，出现问题自动回退，确保应用稳定运行

### 全平台覆盖

✅ **iOS** - 完美支持  
✅ **Android** - 完美支持  
✅ **鸿蒙（HarmonyOS）** - 业内独家第一方支持  
✅ **Expo** - 无缝集成  
✅ **新架构** - 第一时间跟进支持  
✅ **Hermes** - 完整支持字节码格式  

### 简单易用

```js
// 三行代码即可集成
import { UpdateProvider, Pushy } from "react-native-update";

const pushyClient = new Pushy({ appKey });

<UpdateProvider client={pushyClient}>
  <App />
</UpdateProvider>
```

内置多种更新策略，开箱即用：

- 🔕 **静默更新** - 后台自动下载，用户无感知
- 💬 **提示更新** - 友好的系统弹窗提醒
- ⚙️ **自定义策略** - 完全可控的更新流程

### 专业的管理工具

- **命令行工具**：快速发布、版本管理、回滚操作
- **Web 管理界面**：可视化管理，实时查看更新状态
- **CI/CD 集成**：完美融入你的自动化部署流程
- **开放 API**：meta 信息自定义，扩展性极强

## 🎯 适用场景

### 快速修复线上问题
发现 Bug？无需等待应用商店审核，**即时推送修复版本**给所有用户。

### 灵活的版本策略
- **A/B 测试**：向不同用户推送不同版本
- **灰度发布**：先向小部分用户推送，验证无误后全量发布
- **紧急回滚**：发现问题立即回滚到上一个稳定版本

### 降低流量成本
增量更新大幅减少用户流量消耗，尤其适合：
- 内容密集型应用
- 频繁迭代的产品
- 面向海量用户的应用

### 提升用户体验
- 静默更新，用户无感知
- 可自定义更新时机和提示
- 支持后台下载，不影响使用
